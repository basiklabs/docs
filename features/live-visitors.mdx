## Live visitors

The **Live** view in your dashboard shows everyone currently on your site in real time. For each visitor you can see:

- The page they're currently on
- Their device type and browser
- How long they've been in this session
- Whether they've been identified (via `identify()`)

Click any live visitor to follow along in a live replay — you can watch what they're doing right now.

---

## Page views

Every page visit is tracked automatically, including SPA route changes. Tracelit intercepts `history.pushState`, `replaceState`, and the `popstate` event — so React Router, Next.js navigation, Vue Router, and any other client-side router all work without any setup.

What you get:
- Total page views over time
- Unique visitors per page
- Top pages by traffic
- Entry and exit pages

## Session summaries

Every session gets a summary automatically:

| Metric | Description |
| --- | --- |
| **Bounce rate** | Sessions where the user left without interacting |
| **Time on site** | Total active time in the session |
| **Scroll depth** | Average how far the user scrolled across all pages |
| **Pages per session** | Number of distinct pages visited |
| **Device & browser** | Type, OS, browser version |
| **Country & city** | Derived from IP — IP itself is never stored |

## Performance metrics

Tracelit captures Core Web Vitals and load timing for every page, every session:

| Metric | What it measures |
| --- | --- |
| **TTFB** | Time to First Byte — how fast your server responds |
| **FCP** | First Contentful Paint — when the user first sees content |
| **LCP** | Largest Contentful Paint — when the main content is visible |
| **Page load** | Total time until the page is fully interactive |

<img

These are captured per-session and per-page so you can see if a specific page is slow for a specific segment of users. Filter by device, country, or date range to dig in.
